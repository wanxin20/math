● 登录凭证分析结果

  Token 有效期

  JWT Token 有效期为 7 天，配置在 backend/src/modules/auth/auth.module.ts:

  expiresIn: configService.get<string>('JWT_EXPIRES_IN', '7d'),

  后端 JWT Strategy 中 ignoreExpiration: false，会严格校验过期时间。

  当前存在的问题

  凭证失效后的重新登录逻辑未完整实现。 具体缺失：

  1. 无 Refresh Token 机制 — 没有刷新令牌端点，Token 过期后只能重新登录
  2. 前端无 401 拦截处理 — services/api.ts 使用原生 fetch，当 Token 过期后端返回
   401 时，前端不会自动跳转到登录页，用户只会看到请求报错
  3. 无 Token 吊销机制 — 退出登录只清除了 localStorage，服务端无法主动使 Token
  失效

  建议修复

  需要在前端 API 层添加 401 响应拦截，检测到 Token
  过期时自动清除本地凭证并跳转登录页。要我实现这个功能吗？

● 云数据库迁移到服务器本地数据库操作指南

  当前状况

  项目使用 3 个 Sealos 云 MySQL 实例（dbconn.sealosgzg.site），按系统隔离：

  ┌─────────┬────────┬───────────────────────────┐
  │  系统   │ 云端口 │         数据库名          │
  ├─────────┼────────┼───────────────────────────┤
  │ Paper   │ 36594  │ teacher_research_platform │
  ├─────────┼────────┼───────────────────────────┤
  │ Reform  │ 37159  │ teacher_research_reform   │
  ├─────────┼────────┼───────────────────────────┤
  │ Contest │ 41234  │ teacher_research_platform │
  └─────────┴────────┴───────────────────────────┘

  迁移步骤

  1. 在服务器上安装 MySQL

  # Ubuntu/Debian
  sudo apt update && sudo apt install mysql-server -y
  sudo systemctl enable mysql && sudo systemctl start mysql

  # 安全初始化
  sudo mysql_secure_installation

  2. 创建数据库和用户

  CREATE DATABASE teacher_research_platform CHARACTER SET utf8mb4 COLLATE
  utf8mb4_unicode_ci;
  CREATE DATABASE teacher_research_reform CHARACTER SET utf8mb4 COLLATE
  utf8mb4_unicode_ci;

  CREATE USER 'mathapp'@'localhost' IDENTIFIED BY '你的强密码';
  GRANT ALL PRIVILEGES ON teacher_research_platform.* TO 'mathapp'@'localhost';
  GRANT ALL PRIVILEGES ON teacher_research_reform.* TO 'mathapp'@'localhost';
  FLUSH PRIVILEGES;

  3. 从云数据库导出数据

  # Paper 系统
  mysqldump -h dbconn.sealosgzg.site -P 36594 -u root -p
  teacher_research_platform > paper_dump.sql

  # Reform 系统
  mysqldump -h dbconn.sealosgzg.site -P 37159 -u root -p teacher_research_reform
   > reform_dump.sql

  # Contest 系统
  mysqldump -h dbconn.sealosgzg.site -P 41234 -u root -p
  teacher_research_platform > contest_dump.sql

  4. 导入到服务器本地数据库

  mysql -u mathapp -p teacher_research_platform < paper_dump.sql
  mysql -u mathapp -p teacher_research_reform < reform_dump.sql
  # Contest 如果和 Paper 共库，需要单独建库或复用

  5. 修改 .env 文件

  只需改 3 个文件中的数据库连接配置：

  backend/.env.paper
  DB_HOST=localhost
  DB_PORT=3306
  DB_USERNAME=mathapp
  DB_PASSWORD=你的强密码
  DB_DATABASE=teacher_research_platform

  backend/.env.reform
  DB_HOST=localhost
  DB_PORT=3306
  DB_USERNAME=mathapp
  DB_PASSWORD=你的强密码
  DB_DATABASE=teacher_research_reform

  backend/.env.contest
  DB_HOST=localhost
  DB_PORT=3306
  DB_USERNAME=mathapp
  DB_PASSWORD=你的强密码
  DB_DATABASE=teacher_research_platform

  代码本身不需要改动，app.module.ts 中的 TypeORM 配置已经通过 ConfigService
  读取环境变量。

  6. 验证

  # 启动后端验证连接
  cd backend && npm run start:paper
  # 看到日志无数据库连接错误即可

  注意事项

  - Contest 和 Paper 目前共用同一个数据库名
  teacher_research_platform，但云端是不同实例。迁移到同一台服务器后需要给
  Contest 单独建一个库（如 teacher_research_contest），否则数据会混在一起
  - 如果服务器和后端不在同一台机器，DB_HOST 填服务器内网 IP，并确保 MySQL
  允许远程连接（修改 bind-address 和防火墙）
  - 迁移前建议先在服务器上执行 database.sql 建表验证 schema 兼容性，再导入数据
